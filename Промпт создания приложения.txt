Технический промпт для создания веб-приложения Total Cash

Создай веб-приложение для отслеживания текущего финансового положения человека. Приложение должно быть выполнено как современный SPA-интерфейс с очень понятной логикой, но визуально стилизовано под элегантную бухгалтерскую книгу XIX века: благородные бумажные текстуры, тонкие рамки, аккуратная типографика, ощущение дорогого финансового журнала, винтажного реестра и классической финансовой книги. Интерфейс должен быть эстетичным, премиальным, спокойным, не перегруженным.

Общая концепция

Приложение должно помогать пользователю быстро видеть:

Все ежемесячные доходы
Все ежемесячные расходы
Итог по месяцу
Все активы
Все пассивы
Чистый капитал

Экран должен быть разделен на 4 основные области в виде сетки 2x2:

Слева сверху — блок Доходы
Справа сверху — блок Расходы
Слева снизу — блок Активы
Справа снизу — блок Пассивы

Между верхними двумя блоками и нижними двумя блоками должны быть расположены итоговые горизонтальные панели:

Под блоками доходов и расходов — Итоговая сумма в месяц
Внизу страницы под активами и пассивами — Капитал всего = Активы - Пассивы
Функциональные требования
1. Блок «Доходы»

В левом верхнем блоке пользователь может:

добавлять новый источник дохода;
указывать:
название источника дохода;
сумму в месяц;
комментарий / заметку внутри карточки;
редактировать запись;
удалять запись.

Каждая запись должна отображаться как отдельная карточка или строка в стиле старой финансовой книги.

Внутри блока должна автоматически рассчитываться:

общая сумма доходов в месяц.

Примеры источников:

Зарплата
Доход от аренды
Дивиденды
Подработка
Доход от бизнеса
2. Блок «Расходы»

В правом верхнем блоке пользователь может:

добавлять ежемесячный расход;
указывать:
название расхода;
сумму в месяц;
комментарий / заметку;
редактировать запись;
удалять запись.

Внутри блока автоматически рассчитывается:

общая сумма расходов в месяц.

Примеры расходов:

Ипотека
Продукты
Транспорт
Подписки
Образование
Коммунальные услуги
3. Итоговая сумма в месяц

Под верхними двумя блоками должна быть крупная визуальная панель:

Итог за месяц = Доходы - Расходы

Показывать:

общую сумму доходов;
общую сумму расходов;
итоговый баланс;
визуальный индикатор:
положительный итог;
отрицательный итог;
нейтральный итог.

Эта зона должна быть особенно заметной, как центральная строка баланса в бухгалтерской книге.

4. Блок «Активы»

В нижнем левом блоке пользователь может добавлять карточки активов.

Для каждого актива пользователь может указать:

тип актива;
название;
стоимость;
комментарий / описание.

Поддержать типы активов:

Недвижимость
Автомобиль
Банковский счет
Наличные
Инвестиции
Криптовалюта
Бизнес
Другое

Каждый актив отображается в виде отдельной карточки.

Автоматически рассчитывать:

общую сумму активов.
5. Блок «Пассивы»

В нижнем правом блоке пользователь может добавлять карточки пассивов.

Для каждого пассива можно указать:

тип пассива;
название;
сумму обязательства;
комментарий / описание.

Поддержать типы пассивов:

Ипотека
Потребительский кредит
Автокредит
Кредитная карта
Долг физлицу
Налоговые обязательства
Прочее

Автоматически рассчитывать:

общую сумму пассивов.
6. Итоговый капитал

Внизу страницы должна быть крупная итоговая панель:

Капитал всего = Активы - Пассивы

Отображать:

сумму активов;
сумму пассивов;
итоговый капитал;
визуальный статус:
положительный капитал;
низкий капитал;
отрицательный капитал.

Эта зона должна быть самой выразительной на странице.

UX/UI требования
Общая стилистика

Сделай дизайн в эстетике:

элегантных бухгалтерских книг XIX века;
благородной бумаги;
архивных журналов;
дорогих финансовых реестров;
винтажных учетных страниц.

Но при этом интерфейс должен оставаться:

удобным;
современным по UX;
чистым;
минималистичным;
читабельным;
адаптированным для реального использования.
Визуальные особенности

Используй:

теплый светлый фон цвета состаренной бумаги;
легкую фактуру старой бумаги;
тонкие темные или сепия-линии;
аккуратные рамки как у страниц реестра;
типографику с ощущением классической книги;
декоративные, но сдержанные заголовки;
современные поля ввода, стилизованные под рукописные или архивные формы;
карточки с эффектом бумажных вкладок или финансовых записей;
мягкие тени;
цвета: кремовый, слоновая кость, светлая сепия, темно-коричневый, чернильный, приглушенный бордо или темно-зеленый для акцентов.
Атмосфера

Интерфейс должен вызывать ассоциации с:

семейной финансовой книгой;
личным капиталом;
спокойным управлением богатством;
ручным учетом в дорогом архивном журнале.

Не делать дизайн:

слишком темным;
слишком декоративным;
перегруженным;
театральным;
похожим на игру.

Нужен баланс между:

исторической эстетикой;
деловой строгостью;
современным удобством.
Структура интерфейса
Верхняя часть страницы

Сделай аккуратный заголовок приложения:

например: «Финансовая книга» или «Личный баланс»

Под заголовком можно разместить небольшой подзаголовок:

«Доходы, расходы, активы и пассивы в одном реестре»
Основная область

Расположить 4 блока в виде сетки:

2 блока сверху;
2 блока снизу.

У каждого блока должны быть:

заголовок;
кнопка добавления новой записи;
список карточек;
промежуточный итог по блоку.
Итоговые панели

Сделать 2 крупные итоговые панели:

Баланс месяца
Общий капитал

Они должны визуально выглядеть как важные строки сводного баланса.

Логика расчетов
Ежемесячный баланс

Баланс месяца = сумма всех доходов - сумма всех расходов

Общая сумма активов

Активы = сумма всех карточек активов

Общая сумма пассивов

Пассивы = сумма всех карточек пассивов

Капитал

Капитал всего = Активы - Пассивы

Все расчеты должны обновляться автоматически при:

добавлении записи;
редактировании;
удалении.

Доходы

Для каждой записи:

id
название
сумма
комментарий
Расходы

Для каждой записи:

id
название
сумма
комментарий
Активы

Для каждой записи:

id
тип
название
стоимость
комментарий
Пассивы

Для каждой записи:

id
тип
название

комментарий
Поведение интерфейса

Нужно реализовать:

добавление записей через кнопку;
редактирование карточки;
удаление карточки;
плавные анимации появления и удаления;
валидацию числовых полей;
форматирование денежных значений;
поддержку больших чисел;
удобный скролл внутри блоков, если карточек становится много.
Адаптивность

Приложение должно хорошо выглядеть:

на десктопе;
на планшете;
на мобильном экране.

На мобильной версии блоки можно выстроить вертикально в следующем порядке:

Доходы
Расходы
Баланс месяца
Активы
Пассивы
Капитал всего
Технические требования

Сделай приложение как современный фронтенд-проект.

Предпочтительно:

React + TypeScript
либо чистый HTML/CSS/JavaScript, если нужен один файл
структура компонентов должна быть чистой и логичной.

Если используешь React:

сделать компонентную архитектуру;
вынести карточки, панели итогов, формы добавления и layout в отдельные компоненты;
использовать локальное состояние;
можно использовать localStorage для сохранения данных между перезагрузками.
Желательные компоненты
App
Header
SectionCard
IncomeList
ExpenseList
AssetList
LiabilityList
SummaryPanel
ItemCard
EntryModal или InlineForm
Дополнительные пожелания

Добавь:

иконки, стилизованные под финансовые символы;
аккуратные разделители;
ощущение старой книги учета;
тонкие декоративные элементы по краям блоков;
визуально разные акценты для доходов, расходов, активов и пассивов;
деликатную цветовую кодировку итогов.

Можно использовать:

зеленовато-чернильный акцент для доходов и активов;
бордово-коричневый или графитовый для расходов и пассивов.
Что должно быть на выходе

Нужен готовый интерфейс, в котором:

пользователь может вести личный финансовый учет;
все суммы автоматически считаются;
видно месячный денежный поток;
видно размер капитала;
интерфейс выглядит как премиальная бухгалтерская книга XIX века в современном веб-исполнении.

Сделай результат:

красивым;
рабочим;
логичным;
элегантным;
визуально цельным.
